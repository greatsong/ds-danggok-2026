이 수업의 한 줄 정의

한 학기 동안 서울 데이터를 깊이 다룹니다. 기온으로 회귀를, 강남역 혼잡도로 분류를, 지하철역 승차 기록으로 군집을 배워요. 코드 문법은 외우지 않아도 됩니다. AI에게 자연어로 주문하고, 나온 결과가 맞는지 스스로 확인하는 연습이 이 수업의 중심입니다.

🎯 이 수업에서 배우는 네 가지 판단
  • 질문 설정 — 데이터로 답할 수 있는 질문을 세운다
  • 데이터 감식 — 표의 구멍·이상치·편향을 알아챈다
  • 결과 해석 — 모델이 낸 값을 내 말로 설명한다
  • 한계 인식 — 어디까지 믿을 수 있는지 선을 긋는다

📚 데이터 출처

분석은 데이터가 어디서 왔는지 밝히는 데서 시작합니다. 이 교재가 쓰는 데이터의 출처예요.

🔎 이 교재가 쓰는 데이터
  • 서울 기온(회귀, 1~8차시): 기상청 종관기상관측(ASOS) · 기상자료개방포털
  • 서울 혼잡도(분류, 9~11차시): 서울특별시 실시간 도시데이터 · 서울 열린데이터광장
  • 지하철 승하차(군집, 12~14차시): 서울교통공사 제공·서울특별시 배포 · 서울 열린데이터광장
  • 손글씨 숫자(15차시 데모): MNIST · 기계학습 입문 공개 데이터
  • 대부분 공공누리(출처표시) 계열 공개 데이터입니다. 수업용으로 정리한 사본은 modudata에 있습니다.
🔎 이 출처를 믿어도 될까 — 신뢰도를 가리는 4가지

출처 이름을 아는 것에서 한 걸음 더 나아가, 믿을 만한 출처인지 스스로 따져 보는 것이 데이터 감식의 시작입니다. 네 가지만 확인하면 됩니다.

  • 누가 만들었나 — 정부·공공기관·연구기관처럼 책임 소재가 분명한 곳인가
  • 어떻게 모았나 — 표준 장비·공식 절차로 측정·집계했나, 아니면 누군가의 어림짐작인가
  • 공개·검증되나 — 공식 포털에 누구나 확인할 수 있게 공개되고, 이용 조건(라이선스)이 밝혀져 있나
  • 계속 관리되나 — 한 번 올리고 방치된 자료가 아니라, 책임 기관이 정기적으로 갱신·관리하나

이 교재의 데이터는 네 가지를 모두 충족합니다. 기상청·서울특별시·서울교통공사는 각 데이터를 직접 관측·운영하는 국가·공공기관이고, 모두 공식 포털에 공개해 매일·매시간 갱신합니다.
