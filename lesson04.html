🎯 오늘 만드는 것
  • 깨끗해진 추세 그래프 — 믿기 어려운 해를 걸러 낸 뒤 다시 그리기
  • 전처리 판단 기록 — 무엇을 왜 뺐는지 근거와 함께

0. 지난 차시 구멍 2개

  • ① 1950년대 초 = 선이 끊김(1951·1952 값 없음) · ② 1953년 = 뚝 떨어진 점(0.6℃)
  • 오늘 → 이 구멍들의 정체를 밝히고, 어떻게 다룰지 정한다

1. 구멍의 정체 ① — 한국전쟁 결측

🕵️ 1950~1953년이 비어 있는 까닭
  • 1950년 9월~1953년 11월, 한국전쟁으로 서울 관측이 중단됐다
  • 1951·1952년은 유효한 관측이 0일 · 그래서 연평균이 아예 없다
  • 기온이 안 변한 게 아니라 기록하는 사람이 없었다 · "데이터에도 역사가 있다"

2. 구멍의 정체 ② — 1953년 반쪽 관측

🕵️ 1953년 연평균 0.6℃의 진짜 이유
  • 관측이 그해 12월 31일치 하루만 재개됨 · 1953년 관측일은 31일뿐(겨울만)
  • 여름 값이 통째로 빠지니 겨울 평균이 연평균으로 둔갑 → 0.6℃
  • 지구가 추워진 게 아니라 반쪽만 잰 데이터 · 평균이 왜곡된 것

3. 믿을 수 있는 해만 남기기 (①②③)

  • 해결책 = 관측일이 부족한 해를 뺀다 · 기준은 300일(1년의 대부분)
  • 진행 중인 올해도 아직 미완성이니 함께 뺀다
① 샘플 프롬프트
관측일이 300일보다 적은 해는 평균을 믿기 어려우니, 그런 해를 빼고
연평균 추세를 다시 그려 줘. 진행 중인 올해도 빼 줘.
데이터는 여기야: https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul.csv
  • 여러분 화면이 이런 모습이면 성공 · 글자까지 같을 필요 없음
② AI가 준 코드 예시
import streamlit as st
import pandas as pd
import plotly.express as px

st.title("🧹 관측일로 거른 연평균 추세")

url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul.csv"
df = pd.read_csv(url)
df["연도"] = pd.to_datetime(df["날짜"]).dt.year

# 연도별 평균과 '관측일 수'를 함께 구하기
yearly = df.groupby("연도")["평균기온"].agg(["mean", "count"]).reset_index()

# 진행 중인 올해 제외 + 관측일 300일 미만인 해 제외
keep = (yearly["연도"] < df["연도"].max()) & (yearly["count"] >= 300)
clean = yearly[keep]

st.write(f"거르기 전 {len(yearly)}개 해 → 거른 뒤 {len(clean)}개 해")
fig = px.line(clean, x="연도", y="mean", title="관측일 300일 이상인 해만")
st.plotly_chart(fig, use_container_width=True)
  • 깨끗해진 그래프가 뜨면 → 세 박자대로 ③ 개선으로 전후를 비교
③ 프롬프트 개선 예시
좋아! 빼기 전과 뺀 후 그래프를 나란히 놓아서
어떻게 달라졌는지 한눈에 보이게 해 줘.
✔️ 발견 (★) — 무엇이 달라졌나
  • 거르기 전 118개 해 → 거른 뒤 114개 해(1908~2025)
  • 1953년의 0.6℃ 점이 사라지고 추세가 매끈해진다 · 이상치가 직선을 끌어내리던 게 빠짐
  • 데이터를 줄였는데 오히려 더 믿을 만해진다
💡 전처리는 조작이 아니다
  • 학생 질문 "데이터를 마음대로 빼도 되나요?"
  • 핵심 = "믿을 수 없는 값을 빼는 것"과 "불리한 값을 빼는 것"은 다르다
  • 300일 기준은 누구에게나 같은 잣대 · 결과가 마음에 안 든다고 빼는 게 아님 · 그래서 정당하다
📝 수행평가 예고 — 전처리 판단 서술
  • 결측(한국전쟁)과 이상치(1953년 반쪽 관측)를 구분해 원인을 적기
  • 관측일 300일 기준을 왜 정당하다고 보는지 내 말로 설명하기
