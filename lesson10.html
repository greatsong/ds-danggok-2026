정확도만 보면 놓치는 것

🎯 오늘 밝히는 것
  • 혼동행렬 해석표 — 88.8%라는 숫자를 세 각도에서 확인한다

1. 확인 하나 — 아무것도 안 배워도 75%

  • 혼잡 단계 개수: 여유 2,706 · 보통 498 · 약간 붐빔 294 · 붐빔 110 (합 3,608)
  • 여유가 전체의 2,706 ÷ 3,608 = 75.0%
  • 그러면 무조건 '여유'로만 찍는 바보 모델도 정확도 75%다
⚠️ 88.8%를 다시 보자
  • 기준선(전부 여유)이 이미 75% · 내 모델 88.8%는 그 위 13.8%p일 뿐
  • 정확도는 가장 흔한 답만 찍어도 높게 나온다 — 불균형 데이터에선 겉보기와 다르다
🔄 수치는 자란다
  • 혼잡도는 매시간 쌓임 · 개수·정확도·재현율은 수업 당일 다시 확인. 이 교재 숫자는 준비 시점(2026-07-17) 실측

2. 확인 둘 — 정작 '붐빔'을 놓친다 (①②③)

  • 정확도 하나로는 어느 단계를 못 맞히는지 안 보인다 → 혼동행렬과 재현율로 들여다본다
  • 재현율 = 실제 '붐빔' 중에서 모델이 '붐빔'이라 맞힌 비율
① 샘플 프롬프트
지난 시간에 만든 혼잡도 분류기를 이어서 분석해줘. 조건:

- 네 단계를 서로 얼마나 헷갈리는지 혼동행렬을 표로 그려줘.
- 단계마다 재현율을 계산해서, 특히 '붐빔'을 얼마나 잘 잡아내는지
  알려줘.
- 실제로는 '붐빔'인데 다른 단계로 잘못 맞힌 경우가 몇 건인지도
  보여줘.
  • AI가 아래 모습의 코드를 줌 (똑같지 않아도 OK)
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import confusion_matrix, classification_report, accuracy_score

url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul_congestion_log.csv"
df = pd.read_csv(url)

df["시각"] = pd.to_datetime(df["시각"])
df["요일"] = df["시각"].dt.dayofweek
df["시간"] = df["시각"].dt.hour

X = pd.get_dummies(df[["지역명", "요일", "시간"]], columns=["지역명"])
y = df["혼잡도"]
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42)

model = RandomForestClassifier(random_state=42)
model.fit(X_train, y_train)
pred = model.predict(X_test)

# 단계 순서 (표를 보기 좋게 고정)
steps = ["여유", "보통", "약간 붐빔", "붐빔"]

# 혼동행렬 — 어느 단계를 서로 헷갈리나
st.subheader("혼동행렬 (세로=실제, 가로=예측)")
cm = confusion_matrix(y_test, pred, labels=steps)
st.dataframe(pd.DataFrame(cm, index=steps, columns=steps))

# 단계별 정밀도·재현율
st.subheader("단계별 성적표")
st.text(classification_report(y_test, pred, labels=steps, zero_division=0))

# 기준선: 무조건 '여유'로만 찍기
baseline = (df["혼잡도"] == "여유").mean()
st.metric("내 모델 정확도", f"{accuracy_score(y_test, pred):.1%}")
st.metric("전부 '여유'로 찍기", f"{baseline:.1%}")
⚠️ '붐빔' 재현율 = 0.67
  • 정작 대비가 필요한 '붐빔'은 실제 3건 중 1건을 놓친다(재현율 0.67)
  • 여유는 잘 맞히니 전체 정확도는 높다 — 그런데 중요한 단계를 못 맞히면 그 정확도가 무슨 소용인가
③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.
- '무조건 여유로만 찍는 모델'의 정확도를 따로 계산해서
  내 모델 정확도와 나란히 비교해줘.
- '붐빔' 재현율을 크게 강조해서 보여줘.

3. 확인 셋 — 정답을 미리 본 건 아닌지 (레이블 누수)

  • 데이터에 인구min·인구max 열이 있다 · 이걸 특징으로 넣으면 정확도가 88.8% → 90.9%로 오른다
  • 하지만 이건 실력이 아니라 반칙이다
⚠️ 왜 반칙인가 — 레이블 누수
  • 혼잡도는 바로 그 인구 수에서 계산되어 나온 값 — 인구를 알려 주면 답을 미리 보여 주는 셈
  • "혼잡한 시간엔 원래 사람이 많다" → 인구를 넣는 건 시험 전에 답안지를 보는 것과 같다
  • 배포 상황엔 그 시각의 실제 인구를 알 수 없다 — 쓸 수 없는 특징으로 올린 점수는 허깨비

🔎 오늘의 발견 — 정확도만 보면 놓치는 세 가지

✔️ 88.8%만 보면 놓치는 것
  • 하나 — '여유'만 찍어도 75%. 기준선을 안 보면 88.8%가 대단해 보인다
  • 둘 — 정작 중요한 '붐빔'은 3분의 1을 놓친다(재현율 0.67)
  • 셋 — 인구 열로 90.9%까지 오르지만 그건 답을 미리 본 반칙(레이블 누수)
  • → 정확도 한 숫자로 모델을 판단하지 않는다. 무엇을 어떻게 틀리는지까지 본다

📚 오늘의 용어

오늘 나온 낱말 — 외울 필요 없어요 · 잊으면 여기로
용어뜻
기준선(baseline)가장 흔한 답만 찍었을 때의 정확도 — 비교의 출발선
