이미지도 784개의 숫자다 · 팀 짜기

지금까지 다룬 기온·혼잡도·지하철은 모두 표 안의 숫자였다. 오늘은 손글씨 이미지를 열어 본다. 이미지마저 숫자의 묶음이라는 걸 확인하고, 이어서 팀을 짜고 우리 프로젝트 주제를 정한다.

🎯 오늘 하는 것
  • 이미지 데모 — 손글씨 숫자 한 장을 열어 784개의 숫자로 보기 (선생님 시연)
  • 팀 주제 카드 — 2~3인 팀을 짜고 회귀·분류·군집 중 하나를 골라 주제를 적기

1. 이미지도 숫자다 (선생님 시연)

  • 이미지 같은 비정형 데이터도 결국 숫자 표(정형)로 바꿔 다룬다 — 2차시에 본 두 갈래가 여기서 만남

손글씨 숫자 데이터 mnist_small.npz는 숫자 그림 7,000장이 들어 있다. 그림 한 장은 가로 28칸 × 세로 28칸, 즉 784칸이다. 칸마다 0(검정)부터 1(흰색)까지의 밝기 숫자가 하나씩 들어 있다. 그림을 열어 보면 결국 784개의 숫자다.

💡 왜 선생님 시연인가
  • 이번 차시는 "이미지도 숫자"라는 감을 잡는 게 목표. 코드는 선생님 화면으로 함께 본다.
  • 아래 ①②③은 눈으로 따라오면 충분하다. 배포는 하지 않는다.
① 샘플 프롬프트
손글씨 숫자 이미지 데이터(mnist_small.npz)를 스트림릿으로 불러오는 앱을 만들어 줘. 이미지를 하나 골라서 그림으로 크게 보여 주고, 그 그림이 사실은 784개의 밝기 숫자로 이루어져 있다는 걸 나란히 보여 줘.

- 데이터 주소: https://raw.githubusercontent.com/greatsong/modudata/main/data/mnist_small.npz
- 슬라이더로 몇 번째 이미지를 볼지 고를 수 있게
- 그림 한 장은 가로 28칸, 세로 28칸으로 되어 있음
② AI가 준 코드 예시
import io
import urllib.request
import numpy as np
import streamlit as st

st.title("이미지도 784개의 숫자다")

URL = "https://raw.githubusercontent.com/greatsong/modudata/main/data/mnist_small.npz"

@st.cache_data
def load():
    with urllib.request.urlopen(URL) as r:
        d = np.load(io.BytesIO(r.read()))
        return d["X"], d["y"]   # X: 밝기 숫자(7000, 784), y: 정답 숫자

X, y = load()

# 몇 번째 이미지를 볼지 고르기
i = st.slider("몇 번째 이미지를 볼까요", 0, len(X) - 1, 0)

one = X[i]                  # 숫자 784개가 한 줄로
img = one.reshape(28, 28)   # 28 x 28 그림으로 되돌리기

col1, col2 = st.columns(2)
with col1:
    st.write("이 그림은")
    st.image(img, width=180, clamp=True)   # 밝기 숫자를 그림으로
    st.write("정답:", int(y[i]))
with col2:
    st.write("사실은 이런 숫자 784개")
    st.write("가로 28 x 세로 28 =", 28 * 28, "칸")
    st.dataframe(img.round(2))             # 칸마다 밝기 숫자 하나씩
⚠️ 결과가 똑같지 않아도 괜찮아요

②는 "이런 모습이면 성공"이라는 안심용 예시다. AI가 준 코드가 조금 달라도, 그림 한 장이 784개의 숫자로 펼쳐지면 성공이다.

③ 프롬프트 개선 예시
고른 이미지를 컴퓨터가 어떤 숫자로 맞히는지도 보여 줘. 전체를 얼마나 잘 맞히는지 정확도도 함께 알려 줘.
✔️ 시연에서 확인할 것
  • 그림 한 장 = 784개의 숫자. 28 × 28 = 784.
  • 컴퓨터는 이 숫자들만 보고 약 93%를 맞힌다. 우리가 앞서 다룬 표와 똑같이, 이미지도 숫자를 읽는 문제였다.
💼 데이터로 일하는 사람들 (3분)
  • 데이터 분석가·기획자·마케터·연구자 — 오늘 배운 판단(질문·감식·해석·한계)이 그대로 일이 되는 직업들
  • 관심 진로 분야에도 데이터가 있어요 · 팀 주제를 고를 때 내 진로와 잇는 것도 좋은 선택

2. 팀 짜고 주제 정하기

공통 15차시가 끝나면 팀 프로젝트가 시작된다. 오늘 팀을 짜고 주제의 뼈대를 잡는다. 앞서 배운 세 트랙 중 우리 팀 질문에 맞는 틀을 하나 고른다.

  1. 팀 짜기
    2~3인으로 팀을 만든다. 서로 관심 주제가 통하는 사람과 묶이면 좋다.
  2. 틀 고르기
    우리 질문이 숫자를 맞히는 문제면 회귀, 단계·종류를 맞히는 문제면 분류, 정답 없이 비슷한 것끼리 묶는 문제면 군집이다.
  3. 데이터 후보 적기
    답을 데이터로 확인할 수 있는 질문인지 따져 본다. 쓸 만한 데이터 후보를 3개 적는다.
  4. 주제 카드 제출
    아래 항목을 채워 클래스룸에 제출한다.
💡 우리 팀 주제 카드 (이 항목을 채워요)
  • 팀 이름 · 팀원 (2~3인)
  • 우리 질문 한 문장 — 데이터로 답할 수 있고 범위가 명확하게
  • 고른 틀 — 회귀 / 분류 / 군집 중 하나 + 고른 이유
  • 데이터 후보 3개 — 어디서 구할지도 함께
  • 예상되는 구멍 — 이 데이터에 있을 법한 이상치·결측·편향 하나
⚠️ 답할 수 없는 질문은 피하기

"우리 동네는 왜 살기 좋은가" 같은 질문은 데이터로 답하기 어렵다. "우리 동네 지하철역은 아침에 붐비는가"처럼 측정할 수 있고 범위가 분명한 질문으로 바꾼다.

🎒 오늘의 체크

  • 이미지 한 장이 28 × 28 = 784개의 숫자라는 걸 설명할 수 있다.
  • 기온·혼잡도·지하철·이미지가 모두 표 안의 숫자였음을 안다.
  • 팀(2~3인)을 짰다.
  • 회귀·분류·군집 중 우리 틀을 골랐고 이유를 말할 수 있다.
  • 데이터 후보 3개를 적어 주제 카드를 제출했다.
