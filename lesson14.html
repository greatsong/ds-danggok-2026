이름은 사람이 붙인다

🎯 오늘 만드는 것
  • 동네 성격 찾기 앱 — 역 이름을 검색하면 소속 묶음과 그 묶음의 시간대 리듬, 내가 붙인 이름을 보여 주는 앱 (배포까지)

0. 컴퓨터는 묶기만 한다

  • 지난 두 시간, 컴퓨터는 역을 리듬이 비슷한 묶음으로 나눴다.
  • 그런데 컴퓨터가 준 건 묶음 0, 묶음 1, 묶음 2라는 번호뿐이다. 그 번호엔 뜻이 없다.
  • 이 묶음이 "출근하는 동네"인지 "번화가"인지 읽어내는 건 사람의 몫이다. 오늘의 마지막 한 걸음.
⚠️ 묶음 번호는 실행마다 바뀐다
  • k-means의 묶음 번호(0·1·2)는 돌릴 때마다 순서가 달라질 수 있다.
  • 그러니 번호를 외우지 말고, 리듬 그래프의 모양을 보고 이름을 붙인다.

1. 묶음의 리듬 읽기

  • 각 묶음의 평균 시간대 승차 곡선을 그리면 성격이 보인다.
  • 아침에 봉우리가 솟으면 집에서 나서는 동네, 저녁에 솟으면 일하고 노는 동네다.
리듬 모양무슨 동네일까이름 후보
아침(7~9시)에 승차 봉우리아침에 집을 나서는 주거지주거형
저녁(18~20시)에 승차 봉우리일·놀이를 마치고 나오는 곳출근형·번화가형
하루 종일 고른 편사람이 계속 오가는 혼합 지역혼합형
💡 이름의 근거를 말로
  • "강남은 저녁에 승차가 몰리니 번화가형"처럼, 그래프의 모양을 근거로 붙인다.
  • 이름은 정답이 아니라 해석이다. 근거만 또렷하면 다르게 불러도 된다.

2. 검색 앱 만들고 배포하기 (①②③)

  • 만들 것 = 역 이름을 넣으면 소속 묶음 + 묶음 리듬 곡선 + 내가 붙인 이름이 뜨는 앱
  • requirements.txt에는 scikit-learn·plotly만 (기본 설치 제외 · 버전 없이).
① 샘플 프롬프트
지하철역 군집 앱을 배포용으로 정리해 줘. 스트림릿 클라우드에 올릴 거야. 조건:

- 앞서처럼 최신월 데이터를 역별 시간대 '승차 비율'로 바꿔서
  k-means 3묶음으로 나눠 줘.
- 각 묶음의 '평균 시간대 승차 곡선'을 하나의 선 그래프로 겹쳐 그려 줘
  (가로축 시간대, 세로축 평균 승차 비율).
- 위에 역 이름을 입력하는 검색창을 둬. 역을 넣으면 그 역이
  어느 묶음인지 알려 주고, 그 묶음 곡선을 강조해서 보여줘.
- 묶음마다 이름을 붙일 수 있게 해줘. 예: 주거형, 번화가형, 혼합형.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
- 필요한 라이브러리 목록(requirements.txt)도 버전 없이 같이 줘.
② AI가 준 코드 예시
import streamlit as st
import pandas as pd
import plotly.express as px
from sklearn.cluster import KMeans

st.title("🚇 우리 동네 성격 찾기")

@st.cache_data
def load():
    url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/subway_time.csv"
    return pd.read_csv(url)

df = load()

# 전처리 — 최신월, 역별 승차, 시간대 비율
latest = df[df["사용월"] == df["사용월"].max()]
ride_cols = [c for c in df.columns if "승차" in c]
station = latest.groupby("지하철역")[ride_cols].sum()
ratio = station.div(station.sum(axis=1), axis=0).fillna(0)

# 3묶음으로
ratio["묶음"] = KMeans(n_clusters=3, random_state=42, n_init=10).fit_predict(ratio)

# 내가 붙인 이름 — 리듬 곡선을 보고 정한다(번호는 실행마다 바뀔 수 있음)
이름 = {0: "묶음 0", 1: "묶음 1", 2: "묶음 2"}

# 역 검색
q = st.text_input("역 이름을 입력하세요", "강남")
if q in ratio.index:
    cl = int(ratio.loc[q, "묶음"])
    st.subheader(f"{q} → {이름[cl]}")
else:
    st.info("그 역을 찾지 못했어요. 이름을 다시 확인해 주세요.")

# 묶음별 평균 시간대 곡선
시간 = [c.replace(" 승차인원", "") for c in ride_cols]
평균 = ratio.groupby("묶음")[ride_cols].mean().T
평균.index = 시간
긴표 = 평균.reset_index().melt(id_vars="index", var_name="묶음", value_name="평균승차비율")
fig = px.line(긴표, x="index", y="평균승차비율", color="묶음",
              labels={"index": "시간대"})
st.plotly_chart(fig, use_container_width=True)
✔️ 실행하면 이렇게 나온다 (이 달 기준)
  • 강남·홍대입구를 검색하면 저녁에 승차가 몰리는 묶음(번화가형)에 들어간다.
  • 신림·사당을 검색하면 아침에 승차가 몰리는 묶음(주거형)에 들어간다.
  • 곡선을 보고 이름을 주거형·번화가형·혼합형으로 바꿔 붙이면 완성.
③ 프롬프트 개선 예시
좋아! 배포용으로 마무리해 줘.
- 묶음 이름을 주거형·번화가형·혼합형으로 바꿔서 그래프 범례와
  검색 결과에 그 이름이 나오게 해줘.
- 검색한 역의 곡선만 굵게, 나머지 묶음 곡선은 연하게 보여줘.
- 맨 위에 "이 묶음은 컴퓨터가 나눴고, 이름은 사람이 붙였습니다"
  라는 한 줄 안내를 넣어줘.

2-1. 배포하기 (클릭)

  1. 깃허브에 올리기
    main.py와 requirements.txt를 내 깃허브 저장소에 올린다.
  2. 스트림릿 연결
    share.streamlit.io에서 New app → 저장소·main.py 선택 → Deploy. 준비에 1~2분. (화면 구성은 조금씩 바뀔 수 있어요 — 못 찾으면 AI에게 물어보기)
  3. 주소 확인·게시
    생긴 앱 주소(URL)를 갤러리에 올리고, 짝의 앱을 눌러 좋아요와 한 줄 피드백을 남긴다.
💡 지도로도 보고 싶다면 (확장 과제)
  • 지금 데이터엔 역의 위치(위도·경도)가 없어서 지도 색칠은 못 한다.
  • 역 좌표 파일을 따로 구하면 묶음별 색으로 지도에 찍을 수 있다. 여유가 되는 팀의 확장 과제로 남겨 둔다.

3. 군집 트랙 마무리

  • 비지도학습으로 역을 묶고(12차시), 그 묶음이 흔들리는 걸 보고(13차시), 오늘 뜻을 붙였다.
  • 데이터 과학의 마지막 한 걸음은 계산이 아니라 해석이다.
✔️ 세 트랙을 지나며 배운 것
  • 회귀: 직선을 그어도 미래는 늘 의심한다.
  • 분류: 정확도가 높아도 무엇을 놓쳤나를 본다.
  • 군집: 정답이 없기에 사람이 이름을 붙인다.
📝 수행 과제 — 이름의 근거
  • 내가 붙인 묶음 이름 세 개와, 각 이름을 붙인 근거(리듬 그래프의 모양)를 한 줄씩 적는다.
  • 배포한 앱 주소를 갤러리에 올린다.
