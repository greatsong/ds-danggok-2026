배포 + 백테스트

🎯 오늘 만드는 것
  • 혼잡 예측 앱(배포 URL) — 역·요일·시간을 고르면 예상 혼잡 단계와 "얼마나 믿을지"를 보여 주는 앱

1. 정직한 시험 — 안 본 데이터로 재기 (백테스트)

  • 9차시 88.8%는 학습에도 쓴 데이터 근처에서 잰 점수 — 시험 문제를 미리 본 셈일 수 있다
  • 진짜 실력은 학습에 안 쓴 데이터로 재야 한다 = 백테스트
  • 이 데이터엔 시각이 있으니, 최근 1주일을 '미래'처럼 떼어 두고 나머지로만 학습한다
💡 왜 시간으로 자르나
  • 실제로 우리가 궁금한 건 아직 오지 않은 시각의 혼잡도 · 그래서 검증도 "미래"를 흉내 내 뒤쪽 1주로 한다
  • 아무 데나 무작위로 떼면, 같은 시각의 다른 장소가 학습에 섞여 점수가 부풀 수 있다

2. 백테스트 + 배포 앱 (①②③)

requirements.txt
scikit-learn
① 샘플 프롬프트
스트림릿 '서울 혼잡 예측 앱' 하나(main.py)를 만들어줘.
스트림릿 클라우드에 올릴 거야. 조건:

- 데이터는 이 주소에서 불러와(서울 23곳의 매시간 혼잡도 기록):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul_congestion_log.csv
- 시각에서 요일과 시간을 뽑아 특징으로 써.
- 가장 최근 1주일을 검증용으로 떼어 두고, 나머지로만 학습시켜줘.
- 떼어 둔 1주일을 얼마나 잘 맞히는지 정확도로 보여줘.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
- 한국어 라벨, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • AI가 아래 모습의 코드를 줌 (똑같지 않아도 OK)
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

st.title("🚇 서울 혼잡 예측 — 안 본 1주로 검증")

url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul_congestion_log.csv"
df = pd.read_csv(url)

df["시각"] = pd.to_datetime(df["시각"])
df["요일"] = df["시각"].dt.dayofweek
df["시간"] = df["시각"].dt.hour

# 가장 최근 1주일을 '미래'처럼 떼어 두기
cut = df["시각"].max() - pd.Timedelta(days=7)
train = df[df["시각"] <= cut]
test  = df[df["시각"] > cut]

# 장소·요일·시간을 특징으로
X_all = pd.get_dummies(df[["지역명", "요일", "시간"]], columns=["지역명"])
X_train = X_all.loc[train.index]
X_test  = X_all.loc[test.index]

# 나머지로만 학습
model = RandomForestClassifier(random_state=42)
model.fit(X_train, train["혼잡도"])

# 떼어 둔 1주(안 본 데이터)로 정확도 재기
acc = accuracy_score(test["혼잡도"], model.predict(X_test))
st.metric("안 본 1주로 잰 정확도", f"{acc:.1%}")
⚠️ 두 정확도는 다르다
  • 학습 근처에서 잰 정확도와, 안 본 1주로 잰 정확도는 같지 않다
  • 남에게 보여 줄 점수는 안 본 데이터로 잰 것이어야 정직하다
③ 프롬프트 개선 (예시)
좋아! 배포용으로 정리해줘.
- 역과 요일, 시간대를 고르면 예상 혼잡 단계를 알려주는 화면으로 바꿔줘.
- 예측 옆에 "이 예측은 안 본 1주 기준 정확도 ○○%짜리예요" 같은
  '얼마나 믿을지' 문구를 함께 보여줘.
- '붐빔'은 자주 놓칠 수 있으니 참고하라는 안내도 넣어줘.

3. 스트림릿에 배포하기

  • 7차시 기온 예측기와 같은 방식 · 두 번째라 금방 됩니다
  1. 파일 두 개 준비
    main.py와 requirements.txt를 내 GitHub 저장소에 올리기
  2. 배포
    share.streamlit.io 접속 → Create app → 저장소·브랜치·main.py 선택 → Deploy
  3. 주소 공유
    배포가 끝나면 생긴 주소(URL)를 복사 → 갤러리에 게시 · 짝의 앱을 눌러 좋아요와 한 줄 피드백("정말 그런가?" 질문 포함)
🖥️ 화면이 다르게 보이면
  • 스트림릿·GitHub 화면 구성은 조금씩 바뀔 수 있어요 — 버튼을 못 찾으면 AI에게 물어보기("스트림릿 클라우드에 배포하는 지금 방법 알려줘")
