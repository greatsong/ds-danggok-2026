정답표 없이 비슷한 것끼리 묶기

🎯 오늘 만드는 것
  • 지하철역 군집 초안 — 서울 지하철역을 시간대 승차 리듬이 비슷한 묶음으로 나누는 앱

0. 회귀·분류엔 정답이 있었다

  • 회귀: 연도 → 기온이라는 정답이 있었다. 분류: 시간·요일 → 혼잡 단계라는 정답이 있었다.
  • 오늘은 정답표가 없다. 컴퓨터가 스스로 비슷한 것끼리 묶는다. 이것이 비지도학습이다.
종류정답표하는 일예
지도학습있음정답을 보고 규칙을 배움기온 회귀 · 혼잡 분류
비지도학습없음스스로 비슷한 것끼리 묶음오늘: 역 군집

1. 오늘의 데이터 — 지하철역의 하루 리듬

  • 파일: subway_time.csv (2015년부터 매월, 역마다 24개 시간대 승하차 인원 · 약 8만 3천 줄)
  • 역마다 하루 리듬이 다르다. 아침에 사람이 몰려 타는 역이 있고, 저녁에 몰려 타는 역이 있다.
  • 승차 = 지하철을 타는 것. 주거지는 아침에 집을 나서며 타고, 업무·번화가는 저녁에 집으로 가며 탄다.
🔎 데이터 출처 — 서울교통공사
  • 지하철 시간대별 승하차는 서울교통공사가 제공하고 서울특별시가 배포한 자료입니다 · 서울 열린데이터광장 data.seoul.go.kr
  • 공공누리(출처표시) 계열 공개 데이터입니다. 수업용 사본은 modudata에 있습니다.
💡 왜 '비율'로 바꿀까
  • 큰 역은 어느 시간대든 사람이 많다. 승객 수 그대로 재면 리듬이 아니라 크기로 묶인다.
  • 그래서 역마다 하루 총승차를 100%로 놓고 시간대별 비율(정규화)로 바꾼다. 이래야 크기를 지우고 리듬만 남는다.
  • 비율로 바꾸는 이 판단이 오늘의 핵심이다. 다음 시간에 안 바꾸면 어떻게 되는지 직접 본다.

2. 첫 군집 만들기 (①②③)

  • 도구 = k-means(케이민즈): 미리 정한 묶음 수 k개로 비슷한 것끼리 나누는 방법.
  • 잘 나뉜 정도는 실루엣 점수로 잰다. 1에 가까울수록 안이 촘촘하고 묶음끼리 멀다.
  • requirements.txt에는 scikit-learn·plotly만 (스트림릿·판다스·넘파이는 기본 설치 · 버전 없이).
① 샘플 프롬프트
스트림릿 '지하철역 군집' 앱 하나(main.py)를 만들어줘. 스트림릿 클라우드에 올릴 거야. 조건:

- 데이터는 이 주소에서 불러와. 역마다 24개 시간대의 승차·하차
  인원이 매월 들어 있어. 31MB라 한 번 읽고 캐시에 저장해:
  https://raw.githubusercontent.com/greatsong/modudata/main/data/subway_time.csv
- 가장 최근 달만 남겨. 같은 역이 여러 호선에 나오면 승차 인원을 합쳐 줘.
- 시간대별 '승차 인원'만 골라서, 역마다 하루 총승차를 100%로 놓고
  시간대별 비율로 바꿔 줘(크기 말고 리듬으로 묶기 위해).
- k-means로 비슷한 리듬의 역끼리 3묶음으로 나눠 줘.
- 얼마나 잘 나뉘었는지 실루엣 점수를 카드로 크게 보여주고,
  이 점수가 뭔지 한 줄로 설명해 줘.
- 묶음마다 역이 몇 개씩 들어갔는지도 보여 줘.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • AI가 아래 모습의 코드를 준다 (똑같지 않아도 된다).
② AI가 준 코드 예시
import streamlit as st
import pandas as pd
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

st.title("🚇 지하철역 군집 나누기")

# 31MB 파일이라 한 번만 읽고 캐시에 저장
@st.cache_data
def load():
    url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/subway_time.csv"
    return pd.read_csv(url)

df = load()

# 전처리 — 가장 최근 달만, 역별로 승차 합치고, 시간대 '비율'로
latest = df[df["사용월"] == df["사용월"].max()]
ride_cols = [c for c in df.columns if "승차" in c]
station = latest.groupby("지하철역")[ride_cols].sum()
ratio = station.div(station.sum(axis=1), axis=0).fillna(0)

# k-means로 3묶음
model = KMeans(n_clusters=3, random_state=42, n_init=10)
labels = model.fit_predict(ratio)

# 얼마나 잘 나뉘었나 — 실루엣 점수(1에 가까울수록 좋음)
score = silhouette_score(ratio, labels)
st.metric("실루엣 점수 (1에 가까울수록 잘 나뉨)", f"{score:.3f}")

# 묶음별 역 수
st.subheader("묶음별 역 수")
st.write(pd.Series(labels, index=ratio.index).value_counts().sort_index())
✔️ 실행하면 이렇게 나온다 (이 달 기준)
  • 최신월 529개 역, 실루엣 점수 0.46 안팎
  • 묶음 크기: 한 묶음 172역(저녁에 몰려 타는 역), 한 묶음 356역(아침에 몰려 타는 역), 그리고 역이 딱 1개뿐인 이상한 묶음
  • 데이터가 매월 갱신돼 숫자는 조금씩 달라진다. 점수가 0.46 근처면 성공.
③ 프롬프트 개선 예시
좋아! 이어서 고쳐줘.
- 묶음마다 대표 역 몇 개(그 묶음에서 승객이 많은 역)를 함께 보여줘.
- 아침(7~9시)에 몰려 타는 묶음인지, 저녁(18~20시)에 몰려 타는
  묶음인지 한눈에 알게, 묶음별 아침·저녁 비율도 표로 보여줘.

3. 눈에 걸리는 것 하나

  • 정답표가 없는데도 컴퓨터가 역을 묶었다. 신기한 순간이다.
  • 그런데 역이 하나뿐인 묶음이 있다. 왜 한 묶음에 역이 딱 하나일까?
  • 그리고 이런 질문도 남는다. 왜 하필 3묶음일까? 4묶음이면 달라질까?
⚠️ 오늘은 답하지 않는다 (다음 시간 재료)
  • 이 두 질문, 즉 "외톨이 묶음"과 "묶음 수 k의 정답"은 13차시에서 자세히 다룬다.
